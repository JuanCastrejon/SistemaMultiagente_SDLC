These donor-derived specs are intentionally not installed by `templates/manifest.yaml`.

Use them as educational references when a consumer repo needs domain-specific specs:

- `f1-catalogs-admin`
- `f1-security-rbac`
- `f1-stpanel`
- `legacy-security-access`
- `legacy-visual-coverage`

Rule: copy an example only after replacing domain-specific terms, paths, business rules and acceptance evidence with the consumer repo's reality.
