# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 1.1.x | Yes |
| 1.0.x | Critical fixes only |

## Reporting a Vulnerability

Reporta vulnerabilidades de seguridad de forma privada al maintainer del repositorio. Si GitHub Private Vulnerability Reporting esta disponible, usa ese canal. Si no lo esta, contacta al owner del repositorio y evita abrir un issue publico con detalles explotables.

Incluye:

- descripcion del problema,
- pasos de reproduccion,
- impacto esperado,
- version afectada,
- mitigacion sugerida si existe.

## Disclosure Window

El objetivo es responder en 7 dias y coordinar una correccion o mitigacion dentro de una ventana de hasta 90 dias, segun severidad y complejidad.

No publiques detalles tecnicos antes de que exista una version corregida o una mitigacion documentada.
